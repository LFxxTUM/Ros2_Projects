\chapter{Projekt 0.B -- Stereokamera, Objekterkennung und Kalman-Tracking}
\label{ch:projekt0b}

\textbf{Ziel:} Eine simulierte \textbf{Stereokamera} beobachtet einen
\textbf{farbigen Ball}. Wir erkennen ihn im Bild (OpenCV), schätzen seine
\textbf{3D-Position und Entfernung} durch eigene \textbf{Triangulation} und
glätten bzw.\ prädizieren die Bahn mit einem selbst implementierten
\textbf{Kalman-Filter}. Das ist die direkte Vorübung zu Projekt~2
(Tischtennisroboter, Kapitel~\ref{ch:projekt0a} liefert den Bewegungsteil).

\begin{infobox}[title=Was wir übernehmen -- und was wir selbst bauen]
\textbf{Übernommen (Geometrie/Sensor):} ein Kugel-Primitive als Ball und der
Gazebo-Kamerasensor -- kein CAD nötig. \textbf{Selbst implementiert:} die
Farberkennung, die Stereo-Triangulation (Entfernung) und der Kalman-Filter.
Theorie dazu in den Kapiteln~\ref{ch:stereosehen} (Stereosehen) und
\ref{ch:kalman} (Kalman/EKF) -- hier setzen wir sie praktisch um.
\end{infobox}

\subsection{Schritt 3 -- Objekterkennung (OpenCV, HSV)}
In \emph{beiden} Bildern den Ball über seine Farbe finden und den Mittelpunkt
$(u,v)$ bestimmen.
\begin{lstlisting}[style=py,caption={\code{ball\_detector} -- Kern (pro Bild)}]
import cv2, numpy as np
from cv_bridge import CvBridge
bridge = CvBridge()

def detect_ball(img_msg, hsv_lo, hsv_hi):
    img = bridge.imgmsg_to_cv2(img_msg, 'bgr8')
    hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
    mask = cv2.inRange(hsv, np.array(hsv_lo), np.array(hsv_hi))
    cnts, _ = cv2.findContours(mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    if not cnts:
        return None
    (u, v), _ = cv2.minEnclosingCircle(max(cnts, key=cv2.contourArea))
    return float(u), float(v)        # Pixelmittelpunkt
\end{lstlisting}
\screenshotplaceholder{p0b-detection}{Linkes Kamerabild mit eingezeichnetem
Erkennungskreis/Mittelpunkt. Zeigt, dass die HSV-Schwellen den Ball sauber
isolieren.}

\subsection{Schritt 4 -- Entfernung per Triangulation (selbst)}
Bei achsparalleler (rektifizierter) Stereoanordnung ist die Disparität
$d = u_\text{L}-u_\text{R}$. Mit Brennweite $f$ (Pixel) und Basisbreite $B$:
\begin{align*}
Z = \frac{f\,B}{d}, \qquad
X = \frac{(u_\text{L}-c_x)\,Z}{f}, \qquad
Y = \frac{(v_\text{L}-c_y)\,Z}{f}.
\end{align*}
\begin{lstlisting}[style=py,caption={\code{stereo\_distance} -- Triangulation}]
def triangulate(uL, vL, uR, f, B, cx, cy):
    d = uL - uR
    if d <= 1e-6:                    # zu klein -> unendlich/ungueltig
        return None
    Z = f * B / d
    X = (uL - cx) * Z / f
    Y = (vL - cy) * Z / f
    return (X, Y, Z)                 # 3D-Position im Kamerasystem
\end{lstlisting}
$f$ und $(c_x,c_y)$ stammen aus \code{/left/camera\_info}; $B$ ist die in
Schritt~1 gesetzte Basisbreite. Vertiefung: Kapitel~\ref{ch:stereosehen} und
\ref{ch:kalibrierung}.

\subsection{Schritt 5 -- Kalman-Filter (selbst implementiert)}
Lineares Modell mit konstanter Geschwindigkeit (Zustand
$\mathbf{x}=[X,Y,Z,\dot X,\dot Y,\dot Z]^\top$). Prädiktion und Korrektur:
\begin{align*}
\text{Prädiktion:}\quad & \hat{\mathbf{x}}^- = F\hat{\mathbf{x}},\quad
  P^- = F P F^\top + Q\\
\text{Korrektur:}\quad & K = P^- H^\top (H P^- H^\top + R)^{-1},\quad
  \hat{\mathbf{x}} = \hat{\mathbf{x}}^- + K(\mathbf{z}-H\hat{\mathbf{x}}^-)
\end{align*}
\begin{lstlisting}[style=py,caption={\code{ball\_tracker} -- KF-Kern (NumPy)}]
import numpy as np

class KF:
    def __init__(self, dt):
        self.x = np.zeros(6); self.P = np.eye(6)
        self.F = np.eye(6); self.F[0,3]=self.F[1,4]=self.F[2,5]=dt
        self.H = np.zeros((3,6)); self.H[0,0]=self.H[1,1]=self.H[2,2]=1
        self.Q = np.eye(6)*1e-3; self.R = np.eye(3)*4e-4

    def predict(self):
        self.x = self.F @ self.x
        self.P = self.F @ self.P @ self.F.T + self.Q
        return self.x[:3]

    def update(self, z):             # z = gemessene (X,Y,Z)
        S = self.H @ self.P @ self.H.T + self.R
        K = self.P @ self.H.T @ np.linalg.inv(S)
        self.x = self.x + K @ (np.asarray(z) - self.H @ self.x)
        self.P = (np.eye(6) - K @ self.H) @ self.P
\end{lstlisting}
\noindent Reine \code{predict}-Schritte ohne \code{update} liefern die
\textbf{Vorhersage} künftiger Positionen -- die Grundlage für das Abfangen in
Projekt~2.
